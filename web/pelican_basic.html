pelican 介绍——基础篇

Date 2015-08-05 By Windeal.Li Category 前端 Tags pelican / github

pelican是基于python的一个静态博客生成器，可以通过markdown编写md文件，然后用pelican生成html，最后部署在github pages上，作为个人博客。

pelican安装

关于pelican的安装网上已经有很多材料了（也可以查看官方文档pelican官方文档)。 　　pip install pelican markdown 　　pelican和markdown是通过pip工具安装的，这里可能还需要安装pip极其依赖库什么的。我安装的时候比较顺利的，没遇到什么比较大的问题。如果中间有一些没办法安装，看看是不是缺少什么库，一起安装下就好了。这里就不再详细介绍pelican 和 markdown的安装了。

创建生成器项目

前面我们已经安装好了pelican，这里我们要开始使用它了。 　　首先，我们创建一个Blog工作目录，然后快速生成Blog。

mkdir myBlog
cd myBlog
pelican-quickstart

在pelican-quickstart的时候，可能会有一些需要输入的信息，到时根据提示输入就可以了。 　　前面的步骤完成后，我们会看到在myBlog目录下有很多文件和子目录。大概是这样的：

myblog/
├── content              # 存放输入的源文件
│   └── (pages)          # 存放手工创建的静态页面(这个还没有）
├── output               # 生成的输出文件
├── develop_server.sh    # 方便开启测试服务器
├── Makefile             # 方便管理博客的Makefile
├── pelicanconf.py       # 主配置文件
└── publishconf.py       # 发布时使用的配置文件

第一遍文章

在content下用markdown语法编写一篇文章：

Title: pelican+github pages搭建个人静态博客
Date: 2015-07-05
Category: Python
Tags: pelican, github
Slug: my-super-post
Author: Windeal.Li
Summary: Short version for index and feeds

Hello world.

这样我们的博客的第一张site就差不多完成了。我们可以在本机localhost进行测试： 　　先打开http服务器： ./develop_server.sh start 　　然后打开浏览器，输入localhost:8080,可以得到如下图所示：

部署到github上

关于部署到github pages上，可以参考官方文档 Creating Project Pages manually 不过我在自己做的时候有遇到一些问题，所以这里还是依照我最后成功的做法来描述。另外，在github搭建个人博客有两类，一类是直接大家搭建个人主页，每个github帐号只能创建一个，最后github pages的地址是username.github.io（这一类比较简单，网上的大部分教程也是针对这种情况的，本文不详细介绍。） 这里，我们主要介绍另一类情况，用一个repository搭建静态博客，最后得到的github pages地址是username.github.io/projectname ,例如我的是：http://windeal.github.io/myBlog/. 　　首先，登录github网站，在github上创建一个空仓库，如myBlog（这个就不用介绍了吧）。 　　接着就是要把我们的博客搬到github仓库上去了。

cd output
git init
git add .
git commit -m "First commit"

前面这些代码是git的一些简单操作，不明白的可以看下git的用法。 　　我发现，在github上基于repository搭建静态博客时，需要建立在一个新的分支gh-pages，而不能使用master分支，具体原因现在我还没搞清楚。

git branch gh-pages            #创建本地分支
git push origin gh-pages       #将本地分支push到github上
git push -u origin gh-pages    #把我们前面生成的博客（output下面的文件）push到github上

到这里基本的部署就差不多完成了，我们看一下效果。

发现跟本地效果不太一样。 好像主题没有正常工作。 这是因为本地的主题路径的前缀在push之后发生了变化。（比如我们一些图片在本地可能在image目录下，push之后可能就在 windeal.github.io/myBlog/image目录下，所以需要添加地址前缀。 在pelicanconf.py中修改siteurl变量： SITEURL = 'http://windeal.github.io/myBlog/'

插入本地图片

pelican支持插入本地图片。
  1. 创建一个保存静态媒体的目录static， （该目录中可以存放我们的图片）
  2. 在pelicanconf.py中添加一行STATIC_PATHS = [u"static"]， 指定静态目录。
  3. 根据markdown语法在文件中引用该图片。如我们在static中有一图片为test.png，则引用方式为：![imge_name](/imgs/test.png)

Disqus评论插件

  2. 设置testname.disqus.com站点使其允许域名testname.github.io；
  3. 设置以上配置文件为DISQUS_SITENAME = u”testname”，mcsrainbow 为站点ID
